v7.0.0 May 26, 2026 Latest

Ghost Open is the trial.

The headline change is structural. There is no separate trial flow anymore. Install Ghost Architect™ via npm, point it at any codebase, run real work. By the time you hit the paywall, you have already gotten real value, on your own machine, on your own code.

  • + Ghost Open is the trial. Unlimited Question mode, unlimited Recon mode, and four free deep reports across Points of Interest, Blast Radius, Conflict Detection, and Prompt Triage. No signup. No timer. No friction.
  • + Cost transparency in Question mode. Estimate before the call, actual cost after. The same cost preflight that already runs on every paid scan now runs on the free one too.
  • + Profile feature properly gated. --profile, --create-profile, and --list-profiles flags now correctly restricted to Pro, Team, and Enterprise. Open tier sees a clear upgrade message instead of a broken flag.
  • + Worker-driven launch promo. Promo copy is now served dynamically from a Cloudflare Worker at startup. Promo messaging can change without forcing every user to upgrade the CLI. LAUNCH50 is live for the first 100 new or upgrading subscribers: 50% off the first month on Pro, Team, and Enterprise.
  • + Model defaults updated. Default model lineup is now Claude Sonnet 4.6 and Claude Opus 4.7. Significantly lower per-scan API cost in practice. The cost preflight reflects the new pricing automatically.
  • + README rewritten end-to-end. Clearer, more accurate, better structured. If the v6 install instructions confused you, read them again in v7.

v6.0.0 May 20, 2026

License Worker and parser hardening.

v6.0.0 wired the License Worker to the full Stripe subscription lifecycle and shipped a defensive fix for a findings-parser edge case that had been silently skipping output.

  • + License Worker v6.0.0. Stripe subscription lifecycle integration: renewal extends the license, cancellation revokes, plan changes propagate.
  • + Defensive parser fix for findings.json. Edge case where a malformed heading could cause a finding to be silently skipped is now caught with a peek-ahead and a clear error.
  • + Regression test harness. New test suite at tests/parser-fixtures.test.js covers the parser edge cases so the same bug class cannot regress.
v5 highlights April 2026

Context controls, Partner profiles, structured findings.

The v5 series rolled out per-scan context controls, tier-aware context budgets, the Ghost Partner™ profile system for agencies, and a machine-readable findings output for downstream tooling.

  • + Per-scan context flags. --max-context N for raising or lowering the context budget on a single run. --exclude "glob" for path or pattern exclusion. --exclude-presets for curated exclusion bundles (test data, generated files, vendor caches).
  • + Tier-gated context caps. Ghost Open 50K tokens, Pro 100K, Team 150K, Enterprise 200K. The cap is enforced at the loader, not the prompt, so the math is the same on every run.
  • + Ghost Partner™ profile system. Agencies and consultants can save scan configurations as named profiles. --profile name, --create-profile, and --list-profiles flags. Profiles are stored locally per-machine.
  • + Structured Findings JSON. Every scan now emits a machine-readable findings.json alongside the human-readable TXT, MD, and PDF outputs. Downstream tooling, dashboards, and the Ghost Portal report sync all consume this format.
