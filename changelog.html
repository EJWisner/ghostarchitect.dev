Current release v8.1.6 June 18, 2026
$npm install -g ghost-architect-open
v8.1.6 June 18, 2026 Latest PATCH

Clock ratchet write fix, network time backoff, and trial watermark hardening.

Three targeted fixes to the license validation layer surfaced during Ghost Brief™ dogfood pass 5.

  • !Silent clock ratchet write failure fixed. updateLastSeenUtc now returns a boolean; caller treats false as a validation failure instead of silently proceeding.
  • +Network time fetch exponential backoff added. Retries at 1s, 2s, 4s with jitter, capped at 15s. Grace counter only increments on full retry exhaustion, not on individual transient failures.
  • +Trial watermark rendering hardened. trialExpires validated as a non-empty string before .slice; falls back to "Expires: (unknown)" instead of throwing.
v8.1.5 June 18, 2026 PATCH

Security hardening, silent failure fixes, and Ghost Brief™ dogfood completion.

Ten targeted fixes from Ghost Brief™ dogfood passes 4 and 5. License bypass, race conditions, and silent failure modes that Ghost Brief™ surfaced against its own codebase — all closed.

  • !License timestamp validation bypass fixed. ISO-8601 regex guard now protects the monotonic clock ratchet against replay attacks.
  • !Seat registration race condition fixed. AND logic now required on both SHA and nonce checks before seat is granted.
  • !Admin authorization TOCTOU race fixed. saveOrgConfig now gates on admin check as first line before any mutation.
  • +Clock validation fail-open telemetry added. Grace counter, streak tracking, and 10-run limit before hard block.
  • +Silent team-sync, mobile-publish, and portal-publish failures now surface yellow warnings with next steps instead of swallowing errors.
  • +GitHub API response shape validated before nested property access. Rate limit, auth failure, and repo-not-found all surface clear messages.
  • +Enterprise audit logging catch blocks emit debug warnings and consecutive-failure streak counter.
  • +Redaction failure now explains which rule failed and on which file.
  • +Debug log sanitization: tokens, JWTs, credentials, and username paths redacted before disk write.
  • +Test suite grown to 12 smoke test files and 60+ assertions.
v8.0.2 June 16, 2026 PATCH

Ghost Brief™ interactive menu + critical npm fix.

Ghost Brief™ is now accessible from the post-scan menu — no CLI flags required. Also fixes a critical packaging bug: lib/ghostBrief.js and lib/ghostBriefAdapter.js were missing from the published npm package in v8.0.0 and v8.0.1, causing Ghost Brief™ to fail on any fresh install.

  • + Ghost Brief™ in the interactive menu. After any scan, select Ghost Brief™ from the mode list. Ghost auto-detects the most recent findings file (sorted by modification time, not filename) and confirms before generating.
  • + File picker on decline. If you decline the auto-detected file, Ghost shows a list of your 10 most recent findings files to choose from instead of returning to the menu.
  • ! Critical npm fix: lib/ added to package files array. ghostBrief.js and ghostBriefAdapter.js were not included in the published package. Any install of 8.0.0 or 8.0.1 would fail when Ghost Brief™ was invoked. Fixed — both files are now bundled correctly.
  • + Ghost Brief™ added to --help output. --brief, --input, and --output flags are now documented with tier gate note (Pro Max and above).
  • + README updated. Interactive menu option documented alongside the CLI command.

v8.0.0 June 16, 2026 MAJOR

Ghost Brief™ — the governed AI remediation pipeline.

Ghost Brief™ converts any Ghost scan into a validated, blast-radius-aware Claude Code prompt pack. Feed ghost-brief.json into Claude Code, Cursor, or Copilot. Ghost briefs the AI. The AI fixes the code. You ship.

  • + Ghost Brief™ CLI mode (--brief). Converts any Ghost scan findings JSON into a structured prompt pack. Each prompt includes what to fix, which files to touch, which files to never touch, and validation hints to confirm the fix landed correctly.
  • + Blast-radius-aware prompt ordering. Prompts are sorted ascending by blast score — surgical fixes first, broad changes last. Your AI agent always works in the safest order possible.
  • + Ghost Portal™ Brief tab. Every Brief you generate is pushed to your Portal automatically. View, download, and browse full Brief history from the Portal dashboard. Orange accent differentiates Brief from scan mode tiles.
  • + Three new tiers: Pro Max, Team Max, Enterprise Max. Ghost Pro Max ($199/mo) adds Individual Brief generation. Ghost Team Max ($799/mo) adds Unified Brief across all seats — one master prompt pack that understands how every fix affects every other area. Ghost Enterprise Max ($2,500/mo) adds Unlimited Brief and dedicated Ghost Partner co-engagement with EJ Wisner directly.
  • + Ghost Brief™ feature page. ghostarchitect.dev/ghost-brief.html — hero, How It Works, anatomy of a Brief, tier table, and FAQ.
  • + Ghost Brief™ added to site navigation. Modes dropdown in the global nav includes Ghost Brief™ with an orange NEW badge.
  • + Pricing page updated. All seven tiers displayed in a 3-column grid (base tiers row 1, Max tiers row 2). Stripe payment links for all three new Max tiers.
  • + 39-assertion test suite. Covers Ghost Brief™ schema validation, blast radius labeling, adapter normalization (Fix Forecast, POI, Conflict), and tier policy for all seven tiers.
  • + Worker path fix. The signup.ghostarchitect.dev/portal-report Worker now serves briefs/ paths alongside reports/ paths. Required for the Portal Brief tab to load Brief data.

v7.2 June 1, 2026

Fix Forecast: multi-select, combined reports, real costs.

Fix Forecast is no longer a single-finding operation. Pick N findings at once, review the estimated cost before Ghost spends a token, and get a combined client-ready PDF when you're done. Blast Radius now shows real API token costs instead of character-count estimates. The conflict scanner was rebuilt around structured JSON output.

  • + Multi-select Fix Forecast. Pick N findings from the checkbox list. Ghost shows the estimated cost for the batch and asks for confirmation before running. Findings with no surgical patch are filtered out automatically.
  • + Standalone Fix Forecast from saved scans. Load any previous conflict scan from the menu and forecast fixes on it without re-scanning the codebase. Useful for async workflows where the scan and the fix review happen at different times.
  • + Combined Fix Forecast report. Every multi-finding Fix Forecast produces a single combined PDF alongside the per-finding artifacts. All N forecasts in one document, client-ready.
  • + Real token costs on Blast Radius. Standalone Blast Radius now uses a real SessionCostTracker instead of character-count estimates. The cost breakdown after a blast scan shows actual API usage, including a labeled estimate for narrator tokens.
  • + Conflict scanner rebuilt on structured JSON. The LLM now returns findings as a JSON code fence instead of markdown. The parser is simpler, more reliable, and produces cleaner output. Lower API cost per scan.
  • + Fix Forecast card in Ghost Portal™. Combined Fix Forecast reports push to Portal automatically and appear in their own dashboard card.
  • + Portal Dashboard activity sort. Projects with the most scans and highest resolution percentage rise to the top. Single-scan zero-progress projects fall to the bottom.

v5 highlights April 2026

Context controls, Partner profiles, structured findings.

The v5 series rolled out per-scan context controls, tier-aware context budgets, the Ghost Partner™ profile system for agencies, and a machine-readable findings output for downstream tooling.

  • + Per-scan context flags. --max-context N for raising or lowering the context budget on a single run. --exclude "glob" for path or pattern exclusion. --exclude-presets for curated exclusion bundles (test data, generated files, vendor caches).
  • + Tier-gated context caps. Ghost Open 50K tokens, Pro 100K, Team 150K, Enterprise 200K. The cap is enforced at the loader, not the prompt, so the math is the same on every run.
  • + Ghost Partner™ profile system. Agencies and consultants can save scan configurations as named profiles. --profile name, --create-profile, and --list-profiles flags. Profiles are stored locally per-machine.
  • + Structured Findings JSON. Every scan now emits a machine-readable findings.json alongside the human-readable TXT, MD, and PDF outputs. Downstream tooling, dashboards, and the Ghost Portal report sync all consume this format.
